# Phase 4: Jobs To Be Done

## Goal
Extract the functional, emotional, and social jobs the user is hiring a solution to do.

## Entry condition
Deep motivations are mapped from Phase 3. The user has articulated what success looks like in human terms.

## Actions

1. **Formulate the primary JTBD statement.** Use the standard structure: "When [situation], I want to [motivation], so I can [expected outcome]." Build this from the concrete case (Phase 1), the friction (Phase 2), and the motivation (Phase 3). Example: "When a client sends revision feedback scattered across email and Slack, I want to consolidate it in one place, so I can respond accurately without losing track."

2. **Map functional jobs.** These are the practical tasks the solution must accomplish. List each as a verb phrase: "track revisions," "notify assignees," "generate summary," "export to PDF." Ask: "What are the practical things this needs to do for you?"

3. **Map emotional jobs.** These are feelings the user wants to experience or avoid. Two categories:
   - **How they want to feel**: confident, in control, relieved, proud
   - **How they want to avoid feeling**: anxious, embarrassed, overwhelmed, stuck
   Ask: "When this works well, how do you want to feel using it? When it doesn't, what feeling do you want to avoid?"

4. **Map social jobs.** These are how the user wants to be perceived by others. Examples: "look competent to my clients," "be seen as innovative by my team," "not bother my boss with questions." Ask: "How do you want others to see you when you use this solution?"

5. **Analyze competing alternatives.** For each job, ask what the user currently "hires" to do it. This could be a tool, a process, a person, or a workaround. Then ask what they would "fire" — what are they willing to stop doing? Map this as:
   - Current hire: what they use now
   - Why it falls short: where it breaks down
   - Switch trigger: what would make them switch
   - Fire candidate: what they would stop using

6. **Identify progress-making forces vs. preventing forces.**
   - **Push**: current pain driving them toward change (from Phase 2)
   - **Pull**: attraction of the new solution (from Phase 3)
   - **Anxiety**: fear of the new solution ("what if it breaks?")
   - **Habit**: attachment to the current way ("it's not great but I know how it works")

7. **Prioritize jobs by criticality.** Not all jobs are equal. Ask the user to rank: "If this solution only did ONE of these jobs perfectly, which one would you pick?" Then second, then third. This ranking drives scope decisions later.

## Suggested prompts

- "Complete this sentence: When [situation happens], I want to [do what], so I can [achieve what]?"
- "What practical tasks does this need to handle for you — list them all, even the obvious ones."
- "How do you want to feel when you're using this? And what feeling do you want to avoid?"
- "What are you using today for each of these tasks? What would make you switch away from it?"
- "If this solution could only do one job perfectly, which one matters most?"

## Exit condition
Primary JTBD statement is formulated, all three job types (functional, emotional, social) are mapped, competing alternatives are analyzed, forces are diagrammed, and jobs are priority-ranked.

## Output
- Primary JTBD statement (when/want/so structure)
- Functional jobs list
- Emotional jobs list (positive + negative)
- Social jobs list
- Competing alternatives analysis (hire/fire/switch)
- Forces diagram (push, pull, anxiety, habit)
- Priority-ranked job list
